position-static

Elements by default have a position of static. We use this primarily to determine how other attributes affect an element. For example, z-index will not work while an element is positioned static. Why? An element in static is considered reserved and untouched. It is displayed exactly as it is in the box model with regards to the relative height and width to the other elements. Setting other properties will have various effects.

position-relative

This element is positioned relative. Does it look much different? Not exactly, but it does allow us to use some more interesting attributes, as well as affect the content contained within this element in a more precise manner, such as if this text was positioned absolutely. If this element was not positioned relative, and the content was absolute, the text would be in the far left corner of the page. Since, however, this element is positioned relative, the absolute content will rest in the upper left hand corner of this element

position-fixed

This element doesn't care where you are in the scroll. It will always be on the screen. This content belongs to the position-fixed element

position-absolute

This content is positioned absolutely. This content doesn't look much different than the relatively positioned one, but the difference is that the height is no longer evaluated with the default padding. And since the element that contains this element is positioned relatively, it keeps it nice and snug in the enclosing container.
